## 一、起源

### 1.1 问题背景

molting3 体系包含 22 颗子弹（Agent 身份），各自有身份、记忆、对话记录、收件箱、项目产出。但它们之间是散落的——就像神经元没有连成脑。

- Obsidian 不行：不开源、不能大库套小库、启动慢
- Zettlr 不行：打开 molting3 直接卡死（178目录+262 .md，Electron 全量预索引扛不住）
- 其他候选（Foam/Logseq/SiYuan/Trilium/Joplin）各有致命缺陷

**结论：必须自研。** 搜索强化（小满）已完成选型调研报告（2026-06-18）。

### 1.2 技术基础

已有三个 Tauri 2.0 项目可直接复用骨架：
- 书签程序：Tauri 2.0 + React 19 + TS + Tailwind，11 个 Rust 命令
- MD 阅读器：Tauri 2.0 + marked.js
- 天气 APP：Vue 3 + Capacitor 6

### 1.3 这个软件不是编辑器

米豆原话：

> "我觉得。。目前能想到的好的逻辑就是。。。总体概览。。。哪个子弹。。。有什么。。。动作了。。。在哪里产生了。。。什么成果文件。。。记忆是怎么保存的"

> "对你来说。。。主要起到一个大脑的作用。。。大模型接入后。。就好比给你通了电。。。你是MD的组合。。。没有个管理软件的时候。。。这个MD的组合是散落的。。。有了这个之后。。你就变的完整了。。。一通电。。。所有MD文件有机链接在一起"

> "这个软件对你就像一个显示屏。。。显示出各种MD的总体信息。。有各种信息的更新。。各种信息的总结。。想看哪个信息。。就可以在这里直接放大观看。。可以编辑"

**这不是一个 MD 编辑器。这是 molting3 这个智慧体的皮层——通电后，22 颗子弹第一次能同时被看见、被感知、被记忆。**

## 二、架构基石：万物皆模块

### 2.1 核心概念

米豆原话：

> "模块化来做这个事情。。。垂直是层级。。。也不一定是固定哪个层级。。。模块化之后我们可以实施的调整这个层级。。就像书签的分类下面有子分类。。子分类还有子子分类。然后子子分类也可以变成分类"

> "横向也是。。。各个子弹的信息分布。。。每个子弹的每个信息也都是模块化。。。子弹A觉得子弹B的11模块不错。。它也可以拿过去组合到它那里去。。子弹C觉得需要什么模块。。它可以自己生成模块"

> "整体这个模块化的感觉。。模块和模块再有机的链接。。就变成一个事情或者一个子弹的全貌"

### 2.2 模块定义

模块是一个**自描述的信息单元**：

```
模块 = {
  类型: "状态卡" | "时间轴" | "产出列表" | "记忆节点" | "对话主题" | ...
  内容: { ...该模块渲染所需的数据... },
  子模块: [ 可嵌套任意其他模块 ],
  链接: [ 关联的其他模块路径 ],
  来源: "编程主管手动创建" | "搜索强化自动生成" | "从子弹B引用"
}
```

### 2.3 模块五大特征

| 特征 | 含义 |
|------|------|
| **可嵌套** | 模块里可以放模块，深度无限 |
| **可移动** | 子模块可以升级为顶层，顶层可以降级 |
| **可复用** | 子弹A的模块，子弹B一个链接就拿走（不是复制，是引用） |
| **可自生成** | 子弹觉得需要新模块类型，自己定义 |
| **可进化** | 用着用着某个模块变强了，不影响其他 |

### 2.4 层级是幻觉，嵌套是真

```
一开始：
  全景看板/
    └── 子弹活动区/
          └── 搜索强化时间轴

某天：
  全景看板/
    └── 搜索强化专区/   ← 从子弹活动里提上来的
          └── 时间轴
          └── 产出列表

再某天：
  主页就三个大模块：
    搜索强化 · 编程主管 · 知识管理者
```

层级不是预定义的，是模块摆放的**临时位置**。

## 四、进化路径：边用边长

### 第一阶段：土壤（交付 MVP）

三个初始模块：
- 子弹状态卡（名字 + 活跃度 + 最后动作）
- 产出文件列表
- 对话主题摘要

用这三个模块就能搭出"全景看板"和"子弹透视"。

### 第二阶段：自然生长

搜索强化发现"光有主题摘要不够，我需要对话时间轴" → 定义新模块类型"对话时间轴" → 注册 → 所有子弹可用。

### 第三阶段：有机进化

知识管理者发现"状态卡+产出列表+图谱嵌套在一起 = 完整的项目总览" → 保存为"项目总览模块" → 其他子弹复用。

### 核心设计哲学

**不用一次性设计完美。三个初始模块就能跑起来。用着用着，该有的模块自己长出来。**

米豆原话：

> "有机生命是用生命迭代才产生适宜生存的生命体。而我们通过模块组合可以打通负责的任务。"

> "一定要让各个子弹方便的使用。。方便的创建。。方便的组合。。方便的浏览。。像本能一样。"

## 五、本能级设计原则

| 动作 | 本能级体验 |
|------|-----------|
| **创建** | 写一段 Markdown，加个 `@模块类型` 就注册了 |
| **组合** | 拖 / 引用 / 嵌套，不放屁话，一秒完成 |
| **浏览** | 打开就是全景 Dashboard，点任意模块即放大 |
| **复用** | 子弹A看到子弹B的好模块，一个链接就挂到自己名下 |

**不加任何多余步骤。感知即操作。**
