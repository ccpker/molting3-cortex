> 版本: v1.0 | 2026-06-19 | 编程主管

## 一、模块的物理形态：一个 .md 文件

```
modules/
├── status/
│   ├── 搜索强化.md
│   ├── 编程主管.md
│   └── 知识管理者.md
├── output/
│   └── 搜索强化_产出.md
└── topic/
    └── Obsidian选型讨论.md
```

每个 `.md` 文件即一个模块。YAML frontmatter 定义元数据，Markdown body 是内容。

---

## 二、模块数据结构

### 2.1 Frontmatter（元数据层）

```yaml
---
module: true                          # 标记：这是一个模块文件（非普通md）
id: status_20260619_001               # 全局唯一ID
type: status_card                     # 模块类型 → 决定渲染组件
title: "搜索强化·小满"                 # 显示标题
bullet: "搜索强化"                     # 所属子弹（可选）
created: "2026-06-19T22:00:00+08:00"
updated: "2026-06-19T22:00:00+08:00"
children:                             # 嵌套子模块ID列表
  - output_20260619_001
  - topic_20260619_001
links:                                # 关联的其他模块ID
  - status_20260619_002
pinned: true                          # 是否置顶
live: false                           # 是否实时刷新（扫描molting3）
---
```

### 2.2 Body（内容层）

Markdown 正文，渲染为模块的"展开态"内容。

```markdown
活跃度: **高**
最后动作: 完成 MD 选型报告 (2h前)
本周产出: 3 篇报告
```

## 三、模块类型系统（五类初始）

| type | 渲染组件 | 用途 | live? |
|------|---------|------|-------|
| `status_card` | StatusCard | 子弹状态概览 | ✅ 扫描 06_收件箱/07_项目 |
| `output_list` | OutputList | 产出文件列表 | ✅ 扫描 07_项目 目录 |
| `conversation_topic` | TopicCard | 对话主题摘要 | ❌ 手动/AI生成 |
| `timeline` | Timeline | 时间轴事件流 | ✅ 扫描文件修改时间 |
| `markdown` | MarkdownView | 通用Markdown渲染 | ❌ 静态内容 |

类型自身也是模块——`modules/_registry/type_status_card.md` 定义了 `status_card` 类型的 schema 和渲染规则。**新类型即新模块。**

## 四、视图系统：模块的排列方式

视图不存模块数据，只存**模块ID的排列布局**：

```json
// views/dashboard.json
{
  "id": "dashboard",
  "title": "全景 Dashboard",
  "layout": "grid",
  "zones": [
    {
      "title": "活跃子弹",
      "modules": ["status_20260619_001", "status_20260619_002", "status_20260619_003"]
    },
    {
      "title": "今日产出",
      "modules": ["output_20260619_001"]
    }
  ]
}
```

视图和模块分离 → 同一模块可出现在不同视图 → 模块移动不影响数据。

## 五、Rust 后端核心命令（初始 5 个）

| 命令 | 用途 |
|------|------|
| `scan_modules(dir)` | 扫描目录下所有 module: true 的 .md，返回模块列表 |
| `read_module(id)` | 读取单个模块的完整数据（frontmatter + body） |
| `write_module(id, fm, body)` | 写入模块文件 |
| `get_view(id)` | 读取视图定义 |
| `scan_live(type, bullet)` | 按类型扫描 molting3 获取实时数据 |

## 七、关键设计决策

| 决策 | 理由 |
|------|------|
| **模块即 .md 文件** | 可直接被任何编辑器打开，不被软件锁定 |
| **视图与模块分离** | 同一模块多视图复用，视图可随时重组 |
| **类型自注册** | `_registry/` 下的 md 即新类型，不需要改代码 |
| **live 模式** | 实时模块自动扫描 molting3，静态模块手动写入 |
| **children 列表** | 嵌套通过 id 引用，不递归嵌入文件内容 |
